Live Demo

This link is a live demonstration of KOSAR with MySQL and a social networking benchmark that emulates 300 simultanoues users performing interactive social networking actions such as view a member profile, invite a member to be friends, accept a pending friend invitation, list friends, and others.

Why should I care?

The Performance Metrics tab shows the number of queries processed by KOSAR per unit of time. This page is shown at the bottom; the highlighted value is the number of queries processed per unit of time. By turning KOSAR off (using the Setting tab of the live demo), the system process a significantly lower number of queries. When KOSAR is turned back on, the system can process a large number of queries per unit of time.

What else can I do?

The KOSAR interface enables its users to see the query templates that are cached, see the Cached Queries tab of the live demo. KOSAR is caching instance of those templates with unselected radio buttons. By selecting the radio button of a query template and clicking Submit, the user notifies KOSAR to stop caching instances of the selected template(s). Double clicking a selected query template turns its radio button off. By clicking Submit, caching of query instances of the template is re-enabled.

Is there more?

Yes, one may also preview the application's last 100 SQL queries by navigating to the Last 100 Queries tab of the interface. The Last 100 Insert/Delete/Update tab shows the update commands issued by the application. KOSAR caches the result of SQL queries. SQL Insert/Delete/Update commands invalidate the impacted cached query results.
Finally, the SQL Stats tab shows the statistics of the different query templates such as their cache hit rate and their frequency. A query template with a higher frequency constitutes a larger portion of the system workload.

Is this the complete interface of KOSAR?

No, this demo is showcasing the client interface of KOSAR. An application may consist of tens of clients (e.g., Web Servers) and each may employ KOSAR to enhance performance. Each client will have an interface similar to the live demo. These clients are coordinated using KOSAR Core. KOSAR Core has a similar interface that is not demonstrated here.
