White papers

We have organized white papers around topics that shape today's debate about high performance data stores. These include cache augmented middleware, use of smart algorithms, and support for relational data model. The techniques and algorithms described in these white papers are the subject of several patents.

Cache Augmented Middleware

KOSAR is a middleware that enhances the performance of SQL data stores. This middleware is a transparent cache that augments existing systems. An overview of these systems is provided in [1].

1. Cache Augmented Database Management Systems. Proceedings of the Third ACM SIGMOD Workshop on Databases and Social Networks, June 2013.

Intelligent Algorithms

KOSAR embodies patented algorithms that enable it to prevent race conditions that cause the cache to produce stale data [2, 3]. It employs smart cache replacement techniques to enhance the overall system response time and throughput [4].

2. Gumball: A Race Condition Prevention Technique for Cache Augmented SQL Database Management Systems. Proceedings of the Second ACM SIGMOD Workshop on Databases and Social Networks, May 2012.

3. Strong Consistency in Cache Augmented SQL Systems. Proceedings of the ACM/IFIP/USENIX Middleware, December 2014.

4. CAMP: A Cost Adaptive Multi-Queue Eviction Policy for Key-Value Stores. Proceedings of the ACM/IFIP/USENIX Middleware, December 2014

Relational Data Model

A common folklore states, given an application, the relational data model is slow. Pundits argue that switching to a different data model may magically speedup an application. We have investigated this argument with both a document store named MongoDB and a graph data store named Neo4j. With both, we have observed the folklore to be false, see the white papers [5, 6]. It is interesting that normalization guidelines enhance the performance of a relational data store for operations such as finding the shortest distance between two nodes of a graph [5]. Moreover, as long as one manages images intelligently, a SQL solution provides a competitive performance [6].

5. An Evaluation of Alternative Physical Graph Data Designs for Processing Interactive Social Networking Actions. Proceedings of the TPC Technology Conference on Performance Evaluation & Benchmarking (TPCTC), September 2014.

6. A Comparison of Two Physical Data Designs for Interactive Social Networking Actions. Proceedings of the ACM International Conference on Information and Knowledge Management (CIKM), October 2013.
